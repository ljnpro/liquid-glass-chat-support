Release thesis

4.0 is the native architecture era, not a cosmetic patch.

The surface feels cleaner because the internals are cleaner: narrower ownership, explicit recovery, and release gates that keep the app stable as it grows.

Native rebuild

SwiftUI-native from the ground up

GlassGPT moved to Swift, SwiftUI, and SwiftData so the app could behave like a native Apple product instead of a wrapped client.

Modules

Boundaries turned into product leverage

Transport, persistence, runtime, UI, and composition were split into focused targets with enforced boundaries.

Runtime clarity

Recovery and reasoning became explicit systems

Streaming, retry, recovery, and reasoning became named runtime behaviors instead of incidental side effects.

Release quality

Engineering rigor now ships with the interface

Typed throws, accessibility, localization, docs, performance checks, and CI now ship as part of the product contract.

Architecture

The modular design is legible on purpose.

GlassGPT is easier to trust because the seams are obvious: UI stays at the surface, composition wires production once, pure evaluators make decisions, and mutable reply state stays inside the runtime owner.

What users feel

The refactor changed product behavior, not just internal code.

The refactor changed what people feel: better recovery, clearer reasoning controls, steadier routing, and a UI that no longer fights the runtime.

Recovery

Interrupted replies can resume instead of collapsing.

Interruption is treated as a state transition, not a dead end.

  • Recovery fetch, stream, poll, and terminal outcomes are modeled separately.
  • Background and relaunch behavior are handled deliberately.
Reasoning

Reasoning controls now feel integrated instead of bolted on.

Reasoning settings are model-aware and tied more closely to runtime state.

  • GPT-5.4 supports none, low, medium, high, and xhigh.
  • GPT-5.4 Pro focuses on medium, high, and xhigh.
Routing

Direct and gateway routing are now part of a coherent transport model.

Validation can stay direct, gateway routing can stay optional, and first-reply failures are handled more gracefully.

  • Custom gateway URL and token are supported.
  • Direct validation and fallback reduce bad first impressions.
Surface polish

The UI now reflects the underlying discipline.

Settings, history, downloads, and selectors were tightened to feel like one product.

  • Accessibility, localization, and snapshots became normal release work.
  • Generated file preview and share flows now feel native.
Release timeline

How the 4.0 era came together through 4.10.

These milestones are curated from the real changelog. They highlight the shifts that made GlassGPT more native, more modular, and more stable.

4.5 January 20, 2025

The native rewrite becomes real.

The app becomes a native SwiftUI client with SwiftData and package boundaries.

  • Complete native SwiftUI rewrite of the chat client.
  • SwiftData replaces the previous storage path.
  • 16 SPM modules introduce clean dependency boundaries.
4.7 March 18, 2025

Runtime ownership becomes explicit.

Runtime ownership moves into an actor-plus-coordinator architecture.

  • ReplySessionActor becomes the single mutable runtime owner.
  • NativeChatCompositionRoot becomes the sole production composition root.
  • Coordinator-based orchestration replaces broader controller ownership.
4.8 March 18 to March 19, 2026

Engineering rigor turns into a shipping surface.

Quality gates expand from code health into shipping discipline.

  • Typed throws, diagnostics, and MetricKit observability expand the safety net.
  • Accessibility audits, doc coverage, and localization become tracked release requirements.
  • CI grows from basic build checks into a real engineering gate system.
4.9 March 19 to March 20, 2026

Boundaries tighten and release discipline sharpens.

Ownership tightens and release readiness becomes concrete.

  • Settings and history ownership are tightened across application, presentation, and composition.
  • Composition coordinators stop depending on the full ChatController.
  • Localization, documentation, and release-readiness enforcement become standard.
4.10 March 21 to March 23, 2026

The reliability line matures the product around the architecture.

Transport, settings, and runtime hardening converge into the reliability line.

  • Cloudflare gateway configuration modes ship with direct-route validation and fallback handling.
  • Runtime, settings, and CI wiring are stabilized for the 4.10 line.
  • Repeated fixes harden reasoning, recovery, streaming, and settings behavior.
FAQ

Short answers for the practical questions.

Still a support page. Just much tighter.

What does "GlassGPT 4.0" mean on this page? +
It is the umbrella release era, not a single tag. This page groups the native architecture milestones from 4.5 through 4.10 into one product story.
Which models and reasoning levels are supported? +
GlassGPT supports GPT-5.4 and GPT-5.4 Pro. GPT-5.4 offers none through xhigh. GPT-5.4 Pro offers medium through xhigh.
Do I need my own API key? +
Yes. GlassGPT is built around your own OpenAI API key, stored locally on your device.
Is Cloudflare required? +
No. Gateway routing is optional, and direct validation stays available.
What happens if streaming is interrupted? +
Recovery is built into the runtime, so partial replies, reconnects, and relaunch handling behave more predictably.
Privacy

Private by default, explicit by design.

Local storage, explicit credentials, clear routing choices.

  • Local-first storage. Conversations, settings, and most product state stay on device.
  • Credential discipline. API keys are stored in Keychain-backed storage.
  • Direct by default. Gateway routing is optional and explicit.
