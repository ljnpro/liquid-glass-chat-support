Product Highlights

A clearer view of what GlassGPT can do.

GlassGPT is built for people who care about model control, resilient sessions, generated-file handling, and a UI that still feels genuinely native.

Reasoning on your terms

Choose between GPT-5.4 and GPT-5.4 Pro, tune reasoning effort, and keep the app oriented around power-user control instead of one fixed model preset.

Native Liquid Glass styling

The interface is designed around native iOS aesthetics, with Liquid Glass on modern systems and graceful fallback behavior where those materials are not available.

Multimodal input that respects iPhone formats

Send photos with your prompts, including HEIC conversion on iPhone, so multimodal conversations do not feel like a bolted-on web wrapper.

Readable output for technical work

Markdown, LaTeX, code blocks, and streamed reasoning are all presented in a way that supports longer technical sessions instead of just simple chatbot replies.

Recovery, retries, and generated-file resilience

Streaming recovery, stalled response retries, and on-device caching mean both conversations and generated files are more likely to survive interruptions and temporary link expiry.

Private by default, configurable when needed

Your API key and conversation history stay on device. Requests can follow the app's configured OpenAI or gateway route without turning GlassGPT into a data-collecting service.

Changelog

Full Changelog

Release history and in-development notes for GlassGPT.

v3.0.0 March 13, 2026
Major Release — In Development

Cloudflare AI Gateway

  • Replaced WebSocket relay with Cloudflare AI Gateway — All OpenAI API requests now route through Cloudflare’s global edge network (300+ cities) for improved reliability and observability
  • Gateway authentication — Requests authenticated via cf-aig-authorization header to prevent unauthorized gateway usage
  • Gateway health check — Settings UI includes a one-tap connectivity test to verify gateway reachability
  • Zero-maintenance proxy — No server to manage; Cloudflare handles routing, retries, and analytics automatically

Code Interpreter & File Preview

  • Code interpreter output files — AI-generated files (PNG, CSV, etc.) from code execution are now displayed as tappable links in chat
  • Native file preview — Tapping a file link downloads it via OpenAI Files API and opens iOS native QLPreviewController for full-screen preview
  • Share & save — Preview includes standard iOS share sheet for saving files to Photos, Files, or other apps
  • Download progress indicator — Visual progress bar shown during file download with error handling

Swift Concurrency & Build Fixes

  • Swift 6 strict concurrency compliance — Fixed _platformRelayURL global mutable state with NSLock-backed thread-safe storage
  • RelayAPIService Sendable conformance — Added @unchecked Sendable for safe cross-actor usage
  • StreamEvent.completed tuple alignment — Fixed parameter count mismatch after adding FilePathAnnotation support

Architecture

  • Removed WebSocket relay layer — Deleted RelayAPIService.swift and RelaySocketService.swift (1,150+ lines removed)
  • Unified API routing — Single code path for all OpenAI endpoints (/responses, /chat/completions, /files) through configurable base URL
  • New FileDownloadService — Dedicated actor for downloading files with automatic gateway/direct URL resolution
  • New FilePreviewController — SwiftUI wrapper around QLPreviewController for native document preview
  • FilePathAnnotation model — Maps sandbox file paths to OpenAI file IDs for seamless download resolution

Generated File Workflow

  • Dedicated image viewer for png, jpg, and jpeg — Common code interpreter image outputs now open in a cleaner full-screen viewer instead of falling back to generic file handling
  • Dedicated PDF preview — Generated PDFs now open in their own native-feeling reader with sharing support and a cleaner presentation than the old system preview path
  • Direct share for every other output type — CSV, ZIP, JSON, audio, office files, and other generated assets now jump straight into the standard iOS share sheet for handoff to other apps

Caching & Expiring Links

  • Automatic generated-file caching — Files are downloaded and cached on-device as part of the response lifecycle so older chat links are less likely to break when sandbox URLs expire
  • Separate image and document caches — png/jpg/jpeg live in Image Cache while PDFs and all other generated files live in Document Cache
  • Per-cache management in Settings — Each cache shows live usage and can be cleared manually, with the same 250 MB limit applied independently to both buckets

Viewer Polish

  • Theme-aware preview pages — Generated file viewers now follow the app’s system, light, or dark theme selection instead of feeling detached from the rest of the product
  • Better share and save flows — Image preview now feels more deliberate around sharing and saving to Photos, while document-style outputs route directly into the system tools users already expect

Privacy

Your data stays yours

GlassGPT is designed to keep credentials, conversations, and generated files centered on your device instead of turning the app into a data pipeline.

Privacy Policy

  • ✓ No data collection. We do not collect, store, or transmit any of your conversations, messages, or personal information to our servers.
  • ✓ Local storage only. All conversations and settings are stored locally on your device. Your API key is protected using iOS Secure Enclave (SecureStore).
  • ✓ Configurable API routing. Requests follow the app’s configured endpoint path, which can be direct OpenAI or a gateway route such as Cloudflare AI Gateway, without turning GlassGPT into a separate data-collecting backend.
  • ✓ No analytics or tracking. The app does not include any analytics SDKs, advertising frameworks, or user tracking mechanisms.
  • ✓ No account required. You do not need to create an account with us. The only credential needed is your own OpenAI API key.
  • ✓ Generated files remain manageable. Code interpreter outputs can be cached on-device and cleared manually through separate image and document cache controls in Settings.
  • ✓ Endpoint operator policies still apply. Your interactions with OpenAI are subject to OpenAI’s Terms of Use and Privacy Policy. If you use a gateway path, that operator’s policies may also apply.
